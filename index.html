Sistema de Gestión de Pedidos - Análisis de Requisitos

Este documento presenta los requisitos y especificaciones para el sistema de gestión de pedidos de Mary Clean Services, enfocado en digitalizar procesos manuales, mejorar la comunicación con clientes y generar reportes para la toma de decisiones.

Problemas del Negocio

Problemas Actuales

  • Gestión manual ineficiente con pedidos en papel propensos a errores
  • Falta de visibilidad del estado de pedidos, inventario e historial de clientes
  • Comunicación reactiva con clientes mediante llamadas telefónicas
  • Dificultad en facturación y control de pagos con procesos manuales
  • Análisis de negocio limitado por dificultad para generar reportes

Requisitos Funcionales

RF01 - Gestión de Clientes

Crear, buscar, editar y visualizar el historial de clientes.

RF02 - Gestión de Pedidos

Registrar nuevos pedidos, agregar múltiples prendas y servicios, cambiar estados.

RF03 - Gestión de Sucursales

Filtrar información por sucursal en un sistema centralizado.

RF04 - Control de Pagos

Registrar pagos (totales/parciales) y asociarlos a pedidos. Emitir comprobantes.

RF05 - Notificaciones

Enviar notificaciones automáticas por WhatsApp/SMS al cambiar estado de pedido.

RF06 - Gestión de Fotos

Adjuntar fotos de las prendas al momento de la recepción.

RF07 - Generación de Reportes

Generar reportes con filtros por fecha para análisis de negocio.

Requisitos No Funcionales

Usabilidad

Interfaz intuitiva y fácil de usar para empleados con posible resistencia a la tecnología.

Accesibilidad

Debe poder usarse desde celular Android o tablet, ya que puede no haber computadoras.

Costo

Solución de bajo costo o gratuita (open source) inicialmente, acorde al presupuesto limitado.

Conectividad

Debe funcionar con internet inestable o tener modo offline que sincronice después.

Seguridad Básica

Acceso con usuario y contraseña. Diferentes roles (admin, empleado_sucursal).

Entidades Principales

Cliente (id_cliente, nombre, teléfono, dirección, email, fecha_registro)

Sucursal (id_sucursal, dirección, teléfono, ciudad)

Empleado (id_empleado, nombre, id_sucursal, rol, usuario, contraseña)

Servicio (id_servicio, nombre, precio, tiempo_estimado)

Pedido (id_pedido, fecha_creacion, id_cliente, id_sucursal, estado, total, anticipo, fecha_entrega_prometida, fecha_entrega_real)

PedidoDetalle (id_detalle, id_pedido, id_servicio, descripcion_prenda, observaciones, ruta_foto)

Pago (id_pago, id_pedido, monto, metodo_pago, fecha, numero_comprobante)

Relaciones entre Entidades

Cliente - Pedido: 1 Cliente → N Pedidos

Sucursal - Empleado: 1 Sucursal → N Empleados

Sucursal - Pedido: 1 Sucursal → N Pedidos

Pedido - PedidoDetalle: 1 Pedido → N Pedido_Detalles

Servicio - PedidoDetalle: 1 Servicio → N Pedido_Detalles

Pedido - Pago: 1 Pedido → N Pagos (pagos parciales)

Reportes Esenciales

  • Consultas en tiempo real de clientes y estado de pedidos
  • Reporte de caja con ingresos desglosados por método de pago
  • Reporte de pedidos pendientes con días de retraso
  • Reporte de productividad por sucursal
  • Reporte de servicios más populares
  • Reporte de clientes frecuentes para programas de fidelidad

Nota Importante

El atributo ruta_foto en PedidoDetalle es crucial para evitar disputas con los clientes, permitiendo documentar el estado de las prendas al momento de la recepción.

Conclusión para el Desarrollo

El sistema prioritario para Mary Clean Services es un Sistema de Gestión de Pedidos (SGP) centrado en:

  • Digitalizar el proceso de recepción y entrega
  • Mejorar la comunicación con el cliente vía WhatsApp
  • Controlar los pagos y la caja
  • Generar reportes básicos para la toma de decisiones

Una solución inicial podría ser una aplicación web responsive que funcione en celulares, integrada con un API de WhatsApp, y que utilice una base de datos simple en la nube.
